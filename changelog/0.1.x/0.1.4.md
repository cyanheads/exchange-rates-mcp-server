# 0.1.4 — 2026-06-12

## Added

- **`fx_dataframe_query`** — `truncated` boolean output field, `true` when a query exceeds the canvas row cap and the result was capped. `format()` discloses the cap (`⚠️ truncated … refine the query`) and shows `N+ (capped)` in the row count so a capped result is never mistaken for a complete one.
- **`scripts/clean-mcpb.ts`** — post-pack MCPB cleaner wired into `bundle`: runs `mcpb clean` (dev-dependency prune) then strips dependency-shipped agent docs (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) nested under `node_modules/`, which root-anchored `.mcpbignore` patterns cannot reach.
- **`Dockerfile`** — `HEALTHCHECK` (bun-native `fetch` against `/healthz`, no curl/wget in the slim image) and `org.opencontainers.image.version` label fed by an `APP_VERSION` build arg.

## Changed

- **Input-validation errors now classify as `ValidationError` (`-32007`)** instead of `InvalidParams` (`-32602`), tracking the framework's error model in `^0.10.6`. Affects `unsupported_currency`, `date_out_of_range`, and `invalid_range` across `fx_get_rate`, `fx_get_rates`, `fx_convert_currency`, `fx_get_timeseries`, the `invalid_query` reason on `fx_dataframe_query`, and the unsupported-base path on `fx://rates/latest/{base}` (now `validationError` with `base` context).
- **`createApp()`** — explicit `name` and `title` set to `exchange-rates-mcp-server` so the display identity is the hyphenated repo name on every client surface, not the npm scope.
- **`.codex-plugin/plugin.json`** — `interface.displayName` corrected from `@cyanheads/exchange-rates-mcp-server` to the unscoped `exchange-rates-mcp-server`.
- **`scripts/lint-packaging.ts`** — added bundle-content guards (unanchored `.mcpbignore` patterns, critical-runtime-path stripping, post-bundle `node_modules` agent-doc entries) and an identity check that `name`/`title`/`display_name` equal the unscoped package name.
- **`scripts/check-framework-antipatterns.ts`** — new `z.coerce.boolean()` rule (env booleans can't be disabled — `Boolean("false")` is `true`; use `z.stringbool()`) and comment-line filtering so a documented mention isn't flagged as a usage.
- **`.mcpbignore`** — root dev-dir patterns anchored with a leading `/` so `skills/` no longer also strips nested `node_modules/…/skills/` runtime paths.
- Vendored skills resynced to the installed framework version; added the `techniques` skill.
